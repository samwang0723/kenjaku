Architecture Refactor

Make external tool-calling a plugin layer so document RAG, Brave web search, future price quotes, FAQ retrieval, and Agents/Skills all land through one trait — without a single edit to the harness or the brain.

00 Exec Summary — the three decisions that bear weight

Today SearchService::search is a 200-line procedure that owns intent classification, translation, retrieval, web fallback, LLM generation, suggestions, trending, and conversation persistence. Adding a price-quote tool today requires edits in SearchService, AppState, the config loader, and a new trait. The refactor eliminates the 3-4-places edit pain.

01

A single Tool trait

Contract: (ToolRequest, CancellationToken) → Result<ToolOutput, ToolError>. HybridRetriever, BraveSearchProvider, and every future plugin implement it. Self-gating via should_fire.

02

A core Message abstraction

LLM-agnostic message type in kenjaku-core. Providers implement messages_to_wire internally. The ConversationAssembler runs in parallel with tool fan-out. Multi-LLM ready.

03

ToolTunnel DAG executor

Tools declare depends_on(); the ToolTunnel topologically sorts them into tiers at startup (Kahn's algorithm). Within each tier: parallel via join_all. Between tiers: serial, so dependent tools read prior outputs from ToolOutputMap. Per-request CancellationToken cascades on disconnect.

05 The Tool Trait — the crown jewel

Four types + one trait in kenjaku-core. Every existing tier fits this shape; every future tool lands as a single file in tools/. Self-gating keeps ordering dependencies local (Brave's dependence on prior_chunk_count lives with Brave, not in a global router).

/// What the Harness hands a tool on invocation. Owned (not borrowed)
/// so tools can spawn work onto other tasks without lifetime gymnastics.
/// collection_name added in v2 — Harness stays multi-collection-aware.
pub struct ToolRequest {
    pub query_raw: String,
    pub query_normalized: String,
    pub locale: Locale,
    pub intent: Intent,
    pub collection_name: String,
    pub top_k: usize,
    pub request_id: String,
    pub session_id: String,
}

/// Tagged enum. Non-document tools don't shoehorn into chunk shape.
pub enum ToolOutput {
    Chunks    { chunks: Vec<RetrievedChunk>, provider: String },
    WebHits   { hits:   Vec<LlmSource>,     provider: String },
    Structured { facts: serde_json::Value,   provider: String },
    Empty,  // ran, nothing to contribute — Harness won't warn
}

/// Distinct from kenjaku_core::Error so the Harness decides
/// whether to degrade (→ Empty) or propagate (→ user-facing).
pub enum ToolError {
    Disabled,                     // config or rollout
    Timeout(u64),               // millis
    Upstream(String),            // log at warn, degrade
    BadRequest(String),          // propagate as Validation
    Cancelled,                    // SSE client disconnect · NEW in v2
}

/// Per-tool rollout policy. Uniform across every tool;
/// tool-specific knobs (api_key, trigger_regex, etc.) live
/// in the tool's own extension struct.
pub struct ToolConfig {
    pub enabled: bool,
    pub rollout_pct: Option<f32>,  // deterministic hash on request_id
}

use tokio_util::sync::CancellationToken;

#[async_trait]
pub trait Tool: Send + Sync {
    fn id(&self) -> ToolId;
    fn config(&self) -> &ToolConfig;

    /// Is this tool relevant for this request? Self-gating.
    /// Cheap, synchronous, no I/O. Calls config.should_fire_for first,
    /// then layers tool-specific logic (trigger regex, retrieval depth).
    fn should_fire(
        &self,
        req: &ToolRequest,
        prior_chunk_count: usize,
    ) -> bool;

    /// Execute. MUST honor cancel.cancelled() at every I/O await point.
    /// Harness wraps with tokio::time::timeout(tool_budget_ms) as
    /// belt-and-braces for tools that ignore the token.
    async fn invoke(
        &self,
        req: &ToolRequest,
        cancel: &CancellationToken,
    ) -> Result<ToolOutput, ToolError>;

    /// Render Structured payload into a text fact block.
    /// Chunks/WebHits bypass this via [Source N] machinery.
    fn render_fact(&self, facts: &serde_json::Value) -> String {
        serde_json::to_string_pretty(facts).unwrap_or_default()
    }
}
why in-tool gating Gating lives on the tool, not in a central router. Three reasons, all verified against the current code: Brave's trigger already bundles pattern-match + fallback_min_chunks + enabled flag (search.rs:108-120) — self-gating keeps that coupling local; a price-quote tool's trigger is "ticker symbol matches regex" and belongs with the tool; adding an Agent/Skill later = one file, not a router edit.

The ConversationAssembler runs in parallel with tool fan-out.

// Harness error policy: Upstream / Timeout / Cancelled → Empty (degrade).
// BadRequest → fail the request. Disabled → silent skip.
// NEVER use try_join_all — it short-circuits on first error and fights
// the degradation policy.
pub async fn fanout_tools(
    tools: &[Arc<dyn Tool>],
    req: &ToolRequest,
    cancel: &CancellationToken,
    tool_budget_ms: u64,
) -> Result<Vec<ToolOutput>> {
    let futs = tools.iter()
        .filter(|t| t.should_fire(req, 0))
        .map(|t| {
            let t = Arc::clone(t);
            let req = req.clone();
            let cancel = cancel.clone();
            async move {
                match tokio::time::timeout(
                    Duration::from_millis(tool_budget_ms),
                    t.invoke(&req, &cancel),
                ).await {
                    Ok(Ok(out))                       => out,
                    Ok(Err(ToolError::BadRequest(_))) => return Err(t.id()),
                    Ok(Err(_))                      => ToolOutput::Empty,  // degrade
                    Err(_timeout)                    => ToolOutput::Empty,  // degrade
                }
            }
        });
    let outputs: Vec<_> = futures::future::join_all(futs).await;
    // map any BadRequest errors to core::Error::Validation
    Ok(outputs)
}
10 Risks & Decisions — locked in v2

Risks addressed in v2

Risk Severity Mitigation
Latency regression join_all over Vec<Arc<dyn Tool>> ~2-5μs slower per tool than tokio::join!. noise Against 30-300ms tool latency, negligible. ToolRequest clones ~200 bytes × 3 tools × 60 req/min = ~180 allocs/min. Do not micro-optimize back to macros.
Trait object overhead async_trait boxes each invoke call — heap alloc per tool per request. defer Post-Phase-5, if profiling shows it matters, migrate to native async-fn-in-trait (MSRV 1.88 supports it). Punted.
Cancellation propagation Current code drops SSE stream but in-flight Brave HTTP runs to completion. fixing Every Tool::invoke takes &CancellationToken. Harness creates parent token; SSE handler triggers .cancel() on disconnect. tokio::time::timeout as belt-and-braces for tools that ignore it.
Phase 3 prompt fragility Moving build_search_prompt + build_multi_turn_contents out of gemini.rs. canary 50 prompt snapshot tests + 50 messages_to_wire snapshot tests. 10% staging canary for 24h before rollout. Alert on >5% answer-length or refusal-rate delta.
Speculative ToolOutput::Structured No tool emits this today. defended Zero-cost enum variant; Phase 5 tools need it. Kill condition specified: StructuredFact in ToolContext deleted at end of Phase 3 if unused, re-added at Phase 5.
Dependency risk New tokio-util feature for CancellationToken. none Already a transitive dep of tokio::net. Zero new supply-chain surface. No new crates introduced by the refactor.
